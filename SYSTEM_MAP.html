Where the rules live (quality stack)

One principle: each rule lives once.

  • Judgment rules → the system/*.md docs (Architect, Writer, Editor, generators). Rewritten once, cleanly.
  • Mechanical checks → code, so they can't drift: validate-brief.ts (structure, word ceilings, dedup, recency), fact-gate.ts (truth), novelty-gate.ts (Take idea-shape).
  • History → Root_Cause_Library.md + zz_Old/ archives, never inline.

The June compression moved the Architect+Writer+Editor chain from ~87k words → ~12k by enforcing this split.

The improvement loop (the system's immune system)

v2 → brief-critic (mandates + RC1–RC6 tags) → daily-improvement (observe & log) → apply-brief-improvements (weekly consolidation, 1–3 changes, each verified at 7 days) → edits system/ + scripts/ → next brief

Diagnostic half (critic, escalations, root-cause tags, weekly accountability) is the strongest-built part. Prescriptive half is deliberately throttled to weekly so fixes compound instead of thrash.
