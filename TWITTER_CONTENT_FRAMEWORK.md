## Voice & Tone

### Core Principle
**Buddhist teacher at a truck stop.** Detached but helpful. Show the thing, don't explain it.

### NOT:
- Guru voice ("Here's what successful people do...")
- Self-aggrandizing ("We built a lighthouse...")
- Teaching down to people ("Most people don't understand...")
- Claiming victory ("Here's the signal" as if we succeeded)

### YES:
- Humble effort ("I'm trying to figure it out...")
- Genuine invitation ("Maybe you are too...")
- Simple observation ("Here's what I noticed...")
- Personal journey ("Built this for myself...")

## 3-Tweet Thread Structure

Every thread follows this exact structure:

```
Tweet 1: The Scenario (short, punchy, relatable)
Tweet 2: The Book Example (from curated highlights ONLY)
Tweet 3: The Mission (rotate from 12 approved options)
```

## Tweet 1: The Scenario

### Format

```
[Relatable everyday scenario - 2-4 sentences max]

[Model name]

[Link]

Real examples ‚Üí
```

### Rules

- **Concrete, never abstract** - Real situations people experience
- **Everyday scenarios** - Not business jargon or academic examples
- **Let them connect the dots** - Show it, don't explain it
- **No moral. No lesson.** - Just the observation
- **2-4 sentences MAX** for the scenario portion
- **Vary domains** - Don't dominate with one category (cars, food, etc.)

### Length Guidelines

- **Ideal:** 150-200 characters total
- **Maximum:** 280 characters (Twitter limit)
- **Scenario portion:** 2-4 sentences

### Domain Variety

Rotate through different life domains:
- Relationships

- Shopping/Consumer behavior

## Tweet 2: The Book Example

### ‚ö†Ô∏è CRITICAL REQUIREMENT ‚ö†Ô∏è

**ALL highlights MUST come from your curated Readwise highlights.**

- **DO NOT** make up quotes
- **DO NOT** paraphrase from memory
- **DO NOT** use quotes you haven't curated

Each mental model already has curated highlights mapped to it. Use those ONLY.

### Format

```
From [Book Title] by [Author Name]:

"[Shortened version of actual curated highlight]"

```

### Rules

- **Source:** Use ONLY your pre-curated Readwise highlights
- **Shorten if needed:** Keep the quote punchy (aim for 100-200 characters)
- **Maintain integrity:** Don't change the meaning when shortening
- **Attribution:** Always include book title and author
- **Connection:** One sentence that ties the quote back to Tweet 1's scenario
- **Keep it tight:** Total tweet should be under 280 characters

### How to Shorten Highlights

**Original highlight (too long):**
> "The art of economics consists in looking not merely at the immediate consequences of any action or policy, but in tracing the consequences of that policy not merely for one group but for all groups in society, and not merely at the immediate moment but at the longer effects that policy will have."

**Shortened (good):**
> "The art of economics consists in looking not merely at the immediate but at the longer effects of any act or policy."

## Format Rules

### Spacing
- Use line breaks generously
- Makes tweets scannable on mobile
- Easier to read
- Creates visual breathing room

### Emojis
- **Use sparingly:** 0-1 per thread maximum
- **Only if natural:** üßµ for thread indicator is acceptable
- **Never excessive:** No emoji spam
- **Default:** No emojis is better than forced emojis

### Hashtags
- **0-1 per thread maximum**
- **Only if genuinely relevant**
- **Better option:** Zero hashtags, strong content
- **Never spam:** #MentalModels #Thinking #Learning #Growth ‚ùå

### Links
- **Always include in Tweet 1**
- **Format:** `https://mentalmodels.com/model-slug`
- **No URL shorteners:** Use full URLs
- **Consistency:** Same domain every time

### Character Limits
- **Per tweet:** 280 characters (Twitter hard limit)
- **Tweet 1 ideal:** 150-200 characters
- **Tweet 2 ideal:** 200-250 characters
- **Tweet 3:** Use exact approved text (already optimized)

```javascript
// Main thread generation function
function generateThread(model, curatedHighlights) {
  // Select Tweet 3 (favor non-time-based 80% of time)
  const tweet3 = selectTweet3(tweet3Options);
  
  // Get best highlight for this model
  const highlight = selectBestHighlight(model, curatedHighlights);
  
  // Generate thread
  return {
    tweet1: generateTweet1(model),
    tweet2: generateTweet2(highlight, model),
    tweet3: tweet3
  };
}

// Tweet 3 selection (80% non-time, 20% time-based)
function selectTweet3(options) {
  if (Math.random() < 0.8) {
    // Use options 0-9 (non-time-based)
    const humbleOptions = options.slice(0, 10);
    return humbleOptions[Math.floor(Math.random() * humbleOptions.length)];
  } else {
    // Use options 10-11 (time-based)
    const timeOptions = options.slice(10, 12);
    return timeOptions[Math.floor(Math.random() * timeOptions.length)];
  }
}

// Tweet 1 generation
function generateTweet1(model) {
  return `${model.scenario}

${model.name}.

${model.url}

Real examples ‚Üí`;
}

// Tweet 2 generation - MUST use curated highlights
function generateTweet2(highlight, model) {
  // CRITICAL: highlight MUST come from curated Readwise collection
  const shortenedQuote = shortenHighlight(highlight.text);
  const connection = generateConnection(model, highlight);
  
  return `From ${highlight.book} by ${highlight.author}:

"${shortenedQuote}"

${connection}`;
}

// Select best highlight from curated collection
function selectBestHighlight(model, curatedHighlights) {
  // Filter highlights for this specific model
  const modelHighlights = curatedHighlights.filter(h => h.modelId === model.id);
  
  // Sort by relevance and quality scores
  const sortedHighlights = modelHighlights.sort((a, b) => {
    const scoreA = (a.relevanceScore + a.qualityScore) / 2;
    const scoreB = (b.relevanceScore + b.qualityScore) / 2;
    return scoreB - scoreA;
  });
  
  // Return top highlight
  return sortedHighlights[0];
}

// Shorten highlight while maintaining meaning
function shortenHighlight(text, maxLength = 200) {
  if (text.length <= maxLength) return text;
  
  // Find natural break point (sentence, clause)
  const shortened = text.substring(0, maxLength);
  const lastPeriod = shortened.lastIndexOf('.');
  const lastComma = shortened.lastIndexOf(',');
  
  if (lastPeriod > maxLength * 0.7) {
    return text.substring(0, lastPeriod + 1);
  } else if (lastComma > maxLength * 0.7) {
    return text.substring(0, lastComma) + '...';
  } else {
    return shortened + '...';
  }
}

// Generate connection sentence
function generateConnection(model, highlight) {
  // One sentence tying highlight back to scenario
  // Keep under 50 characters
  // Examples:
  // "Consequences of consequences."
  // "Every yes is a no to something else."
  // "Most people aren't out to get you. They're just winging it."
  
  return generateShortConnection(model, highlight);
}
```

### Integration with Existing System

```javascript
// Load data from your existing files
import { allMentalModels } from './lib/readwise-data';
import { curatedHighlights } from './lib/parse-all-domains';

// Generate threads for all models
function generateAllThreads() {
  const threads = [];
  
  for (const model of allMentalModels) {
    // Get curated highlights for this model
    const modelHighlights = curatedHighlights.filter(h => 
      h.modelId === model.id || h.modelId === model.slug
    );
    
    if (modelHighlights.length === 0) {
      console.warn(`No highlights found for ${model.name}`);
      continue;
    }
    
    // Generate thread
    const thread = generateThread(model, modelHighlights);
    threads.push({
      ...thread,
      modelId: model.id,
      modelName: model.name
    });
  }
  
  return threads;
}

// Export for scheduling/posting
export { generateAllThreads, generateThread };
```

## Posting Strategy

### Frequency
- **2-3 threads per day**
- Space them out for maximum reach
- Consistency matters more than volume

### Optimal Times (EST)
- **Morning:** 8-9am
- **Lunch:** 12-1pm
- **Evening:** 5-6pm

### Weekly Strategy
- **Monday-Friday:** Focus on work-related models (constraints, opportunity cost, decision-making)
- **Weekends:** More lifestyle/personal models (relationships, health, learning)
- **Vary domains:** Don't post similar scenarios back-to-back

### Monthly Planning
- Generate 60-90 threads (2-3x daily for 30 days)
- Review for quality before scheduling
- Track which models/scenarios perform best
- Adjust future generation based on engagement

### Content Mix
- **50%** - Common experiences (everyone relates)
- **30%** - Work/professional scenarios
- **20%** - Niche but insightful scenarios

## Key Principles

### Never Forget These

1. **Show, don't tell**
   - Let them discover the model through the scenario
   - No explicit teaching or explaining
   - The "aha" moment comes from them

2. **Humble effort, not victory**
   - We're trying, not succeeding
   - It's a journey, not a destination
   - "Maybe it helps you" not "This will help you"

3. **Invitation, not command**
   - "Maybe you are too" not "You should"
   - "See if it clicks" not "This will change your life"
   - "Use what helps" not "You need this"

4. **Concrete, not abstract**
   - Real situations people experience
   - Specific examples, not general concepts
   - "Quarter tank, middle of nowhere" not "When evaluating risk..."

5. **Buddhist teacher energy**
   - Detached helper, not guru
   - Pointing at the moon, not the moon itself
   - They figure it out, you just show the way

6. **Two truckers talking**
   - Equals sharing insights
   - Not expert to student
   - "Huh, noticed this thing" not "Let me teach you"

7. **Short and punchy**
   - Respect their time and attention
   - Every word earns its place
   - 2-4 sentences for scenarios

8. **Rotate domains**
   - Keep scenarios varied and fresh
   - Don't dominate with one category
   - Relationships, work, money, health, etc.

9. **Real book quotes ONLY**
   - Use your actual curated highlights
   - Never make up quotes
   - Maintain integrity always

10. **No self-aggrandizing**
    - Ever. Period.
    - It's an attempt, not a solution
    - Built for me, might help you
